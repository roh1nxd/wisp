The process

From sentence to live contract

Three steps, in this exact order — each one gates the next, so nothing reaches mainnet untested.

01

Describe it

Type your idea in plain language. Wisp asks a couple of clarifying questions — refund window, fees, roles — and turns it into a spec.

02

Generate & test

Wisp matches an audited Soroban template, fills it in, and runs it through the local sandbox test suite before you see it.

03

Deploy

Ships to testnet by default. Connect a wallet and promote to mainnet only once your tests are green.

Guardrails

Non-technical shouldn't mean unsafe

Financial contracts with subtle bugs cause real losses across the industry. Wisp is built around defaults that make that harder to do by accident.

Default

Testnet first, always

Every project deploys to testnet automatically. Mainnet is a separate, explicit step.

Gate

Tests must pass to promote

The generated test suite has to go green before Pulsar will let you deploy to mainnet.

Base

Templates, not blank slate

Contract logic starts from a reviewed template. The AI fills in parameters, not core logic.

Visible

Unaudited is labeled unaudited

Anything outside the template library ships with a visible warning until a human reviews it.
